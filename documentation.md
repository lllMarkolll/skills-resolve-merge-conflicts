# Konfigurationsmanagement – Übung: Pull Request Review & Merge Conflicts

## Überblick
Im Rahmen dieser Übung wurden zwei zentrale Tätigkeiten aus dem Konfigurations- und Versionsmanagement durchgeführt:
1. Review eines Pull Requests  
2. Auflösen eines Merge Conflicts

Ziel war es, praktische Erfahrung im Umgang mit kollaborativer Softwareentwicklung und Versionskontrolle zu sammeln.

---

## Übung 1: Review eines Pull Requests

### Vorgehen
- Erstellen und öffnen eines Pull Requests im Repository
- Analyse der vorgeschlagenen Code-Änderungen
- Überprüfung auf:
  - Funktionalität
  - Lesbarkeit und Struktur
  - Einhaltung von Konventionen
- Abgabe von Feedback (Kommentare bzw. Freigabe)

### Erkenntnisse
- Pull Requests ermöglichen eine strukturierte Qualitätskontrolle vor dem Merge
- Reviews helfen, Fehler frühzeitig zu erkennen und Codequalität zu verbessern
- Klare Commit-Messages und saubere Code-Struktur erleichtern das Review erheblich
- Feedback sollte präzise, sachlich und nachvollziehbar formuliert sein

---

## Übung 2: Resolve Merge Conflicts

### Vorgehen
- Simulieren bzw. Öffnen eines Merge Conflicts
- Identifikation der konfliktbehafteten Stellen im Code
- Vergleich der konkurrierenden Änderungen
- Entscheidung, welche Änderungen übernommen werden
- Manuelles Auflösen des Konflikts
- Abschließender Commit der bereinigten Version

### Erkenntnisse
- Merge Conflicts entstehen häufig durch parallele Änderungen an denselben Code-Stellen
- Ein gutes Verständnis des Codes ist notwendig, um fundierte Entscheidungen zu treffen
- Regelmäßiges Pullen und kleinere Commits reduzieren die Wahrscheinlichkeit von Konflikten
- Versionskontrollsysteme kennzeichnen Konflikte klar, erfordern aber manuelle Entscheidungen

---

## Fazit
Die Übung hat gezeigt, wie wichtig strukturierte Zusammenarbeit, klare Kommunikation und sauberes Versionsmanagement in der Softwareentwicklung sind.  
Sowohl Pull Request Reviews als auch das Auflösen von Merge Conflicts sind essenzielle Bestandteile professioneller Entwicklungsprozesse.
